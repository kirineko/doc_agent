## Context

Doc Agent 在首轮对话 `turn_complete` 后通过 `maybe_autotitle_session` 自动命名仍保持默认标题的会话。当前 `summarize_session_title` 优先取助手首行（上限 24 字），侧栏展示时常出现寒暄或完整陈述句。右侧 `ProjectFileExplorer` 在非根目录时于标题栏右上角展示 10px 的 `..`，与文件列表分离，可发现性弱。

用户已确认决策：标题为用户意图摘要、16 字上限；泛化开场保持「新会话」并在第二轮重试；文件浏览采用 Finder 风格列表首行返回 + 面包屑，根目录用 `⌂` 符号（非文字）。

## Goals / Non-Goals

**Goals:**

- 自动标题反映用户意图，侧栏可快速区分会话
- 泛化首轮不强行命名；第二轮给实质提问一次补命名机会
- 文件浏览提供明示「返回上级」与可点击面包屑（`⌂` 为根）
- 保持现有 IPC 与单层目录浏览模型不变

**Non-Goals:**

- LLM 生成标题（额外请求与成本）
- 第三轮及以后继续自动重试
- 用户手动改名后的覆盖或合并逻辑变更
- 完整文件树、拖拽、重命名、路径手动输入
- 引入新图标库（沿用 emoji/Unicode 与现有 Tailwind 样式）

## Decisions

### D1：标题来源与字数

- `MAX_TITLE_CHARS`：24 → **16**
- 优先级：**用户消息** → 助手首行（仅首轮且用户无法产出标题时作兜底，须过滤寒暄/自我介绍）
- `summarize_session_title` 返回 `Option<String>`：`None` 表示跳过写入

### D2：泛化开场白检测

命中则本轮不生成标题（保持默认）：

- 固定短语（去标点后大小写不敏感）：`你好`、`您好`、`嗨`、`在吗`、`你是谁`、`介绍一下`、`hi`、`hello`、`hey`
- 或：去标点后 ≤3 字，且不含文件名模式、项目代号（如 `SK\d+`）、任务动词（`分析/总结/转换/导出/合并/查看/读取` 等）

`clean_intent` 在现有 `clean_question` 基础上保留文件名与代号，去掉礼貌前缀后截断为意图短语。

### D3：自动标题触发窗口

`maybe_autotitle_session` 仅在标题仍为默认（`新会话` / `会话 N`）时运行：

| `user_count` | 行为 |
|--------------|------|
| 1 | 用当前轮 `user_text` + 可选 `assistant_text` 尝试命名 |
| 2 | **仅**用当前轮 `user_text`（第二条用户消息），**不用**助手回复 |
| ≥3 | 不再尝试 |

第二轮是唯一延期重试点，覆盖「先寒暄、后提需求」路径。

### D4：文件浏览 — Finder 返回行

- 非根目录时，列表**第一项**为「返回上级」（文案固定，样式与文件行区分，如浅底或虚线框）
- 删除标题栏右上角 `..` 按钮，避免重复入口
- 点击行为复用现有 `parentPath(currentPath)` + `loadDir`

### D5：文件浏览 — 面包屑

- 路径行展示：`⌂` + 各级目录名，以 `/` 分隔
- `⌂` 表示项目根，**可点击**回到 `.`；`aria-label` / `title` 为「项目根目录」（符号无文字时的无障碍补偿）
- 非当前段的目录名可点击跳到对应前缀路径；当前段高亮且不可点
- 根目录仅显示 `⌂`（当前位置）
- 过长路径：`⌂ / … / {末段}`，保留根与当前父级可点（实现时按 `w-64` 窄栏 truncate）

### D6：符号选择

根节点采用 Unicode `⌂`（U+2302），与现有 📁/📄 emoji 共存且更克制。若渲染异常再 fallback 为 `🏠`（实现阶段目检即可）。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 启发式标题仍不够「准」 | 16 字 + 保留文件名/代号；用户可手动改名（后续能力） |
| 第二轮仍泛化则永久「新会话」 | 符合预期；仅两轮窗口，避免骚扰 |
| `⌂` 在小字号下不清晰 | `title`/`aria-label` 补语义；必要时 fallback |
| 面包屑与返回行功能重叠 | 返回行服务「上一级」肌肉记忆；面包屑服务跳多层 |

## Migration Plan

1. 合并后新规则仅影响**之后**完成的 turn；已存在会话标题不回溯改写
2. 无数据库 schema 变更
3. 回滚：还原 `session_title.rs` 与 `ProjectFileExplorer.tsx` 即可
